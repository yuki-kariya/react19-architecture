# 2. Adopt Codebase Conventions

Date: 2026-05-31

## Status

Proposed

## Context

時間が経つにつれ、各モジュールやコンポーネントの依存フローを追跡することが困難になり、メンテナンスコストが増大する。

その理由としてソースベースを確認した時に、どこにどんな処理が記述されているのか一目見てわからず、関心のあるコードに辿り着くまでに時間がかかってします。

## Decision

ルールを文書化する。

アーキテクト、エンジニアはこのルールを把握した上でコードを書かなければならない。

また、本ルールはCodexやClaude CodeといったAIエージェントに特定SKILLとして読み込ませる。

## Consequences

文書化されたルールを事前に把握しておくことで、書き手側は大きなプロセスの塊を論理的に分割でき保守性に優れたコードの記述が可能になり、読み手側は真に関心があるコード箇所に素早く到達できるようになる。

AIエージェントがコードの自動生成をする際も統一されたルールに従うため、コードの品質と、AIと人間による相互レビューの品質を向上させる。

事前にルールを文書化することで、コードの書き方に関してエンジニア同士が衝突することを未然に防ぐことができる。
